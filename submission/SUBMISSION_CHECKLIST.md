# Final Kaggle Submission Checklist

## Eligibility requirements

- [ ] Every collaborator accepted the Kaggle team invitation before the
  applicable team-lock/submission deadline.
- [ ] One submission owner is designated; no teammate will submit a duplicate.
- [ ] Kaggle Writeup has a title, subtitle, and detailed analysis.
- [ ] **Agents for Business** track selected.
- [ ] Writeup is no more than 2,500 words.
- [ ] Media Gallery contains a required cover image.
- [ ] Public YouTube video is attached and no more than 5:00.
- [ ] Public project link is attached and works without login/paywall.
- [ ] Public GitHub repository is attached with detailed setup instructions.
- [ ] Final **Submit** button clicked; project is not left as a draft.

## 30-point pitch evidence

- [ ] Problem explains business cost/revenue/risk impact.
- [ ] Why agents are necessary is explicit.
- [ ] One-sentence value proposition appears in video and writeup.
- [ ] Architecture is shown visually.
- [ ] Video demonstrates the real agent working.
- [ ] Build journey and technologies are mentioned concisely.

## 70-point implementation evidence

- [ ] Root ADK orchestrator and six specialists visible in code/diagram.
- [ ] MCP call visible in Agent Trace.
- [ ] Security: blocked injection, confined upload, secret hygiene, HITL gate.
- [ ] Docker/Cloud Run reproduction documented.
- [ ] Mission rubric proves objective-specific completion.
- [ ] Cross-session `LoadMemoryTool` proof recorded.
- [ ] Pending → rejected → revised → approved report flow recorded.
- [ ] Ten routing cases run and judge-evidence JSON downloaded.
- [ ] GitHub Actions CI is green.
- [ ] README contains problem, solution, diagram, setup, testing, security,
  deployment, and repository map.
- [ ] `SECURITY.md`, `CONTRIBUTING.md`, architecture, evaluation methodology,
  and team collaboration guide are present.

## Privacy and quality

- [ ] No API key, password, `.env.docker`, private email, or admin credential in
  Git, screenshots, video, console output, or writeup.
- [ ] No `.venv`, database, upload, report, export, or bytecode committed.
- [ ] All URLs tested in an incognito window.
- [ ] Evaluation numbers copied from verified evidence—not estimated.
- [ ] Limitations stated honestly.
- [ ] Final video audio, text size, and 1080p processing checked.
- [ ] Team names, Kaggle profiles, GitHub handles, and contribution credits are
  accurate and approved by every collaborator.
